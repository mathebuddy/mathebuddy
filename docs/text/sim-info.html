This page enables to simulate courses. Since the simulator is written with Flutter, the behavior is identical to the app.

For a simulation of parts of the app, choose "PLAY" from the main menu.

Explore the demo course

Remark: The demo course is currently not editable.

The demo course demonstrates how to write courses in the mathe:buddy language (MBL). Each file concentrates on a separate technological aspect.

Steps:
  1. Click on the [DEMO] button above.
  2. Choose file HELLO.MBL. You will see the MBL code on the right info page.
  3. Optionally click on [MBCL] to see the compiled code, that is transferred to the simulator.
  4. Check in the log area (below the simulator window) that no error occurred.
  5. Start the simulator by pressing on the power on button . The current course (or level) can be opened in the simulator window by selecting DEBUG.
  6. Optionally change to another screen size by pressing on one of the smartphone buttons .

Create own courses

Remark 1: The following guide will currently not work with all browsers (e.g. not in Apple Safari). As a workaround, use another browser (e.g. Firefox) or run this website locally on your computer, as described here in the readme file.

Remark 2: Windows Users may find more detailed instructions below.

Creating and editing own courses requires to store MBL files locally on your device. As a dependency you need to have Python 3 installed on your computer.

Steps:
  1. Change to the directory, where you like to store your course data (or download and extract the example course form here and change into the extracted directory).
  2. The example course already contains the file server.py. Run python3 server.py. The script runs a local webserver at port 8271. (If you created your own directory, get the server file from here).
  3. On this website, click on the [HTTP://LOCALHOST:8271] button above. You should see all *.mbl files and directories.
  4. Start the simulator by pressing on the power on button . The current course (or level) can be opened in the simulator window by selecting DEBUG.
  5. Edit the course files on your local computer and press again to observe the changes.
  6. Hint: you should debug level files first, and then debug complete chapters afterwards!

Create own courses in an Windows Environment

The following workflow describes how to edit courses in Windows (... using an UNIXoid operating system is more recommended). We use "GitHub Desktop" to synchronize course data, Visual Studio Code as editor and Python as file server.

Other toolchains may also work, e.g. using Git on the command line. Used Editors MUST save files as plain text, i.e. without formatting (whoever tries to edit course description files with Microsoft Word is disqualified to contribute to mathe:buddy for at least a life time...).

(I.) Installation of Dependencies
Note: You may need to run installers with admin rights.

Python
  • Download Python from https://www.python.org/downloads/ and run the installer.
  • Choose "add python.exe to PATH" in the installer.
  • Choose "disable path length limit" at the end of the installation (last installer panel).
Visual Studio Code
  • Download Visual Studio code from https://code.visualstudio.com/Download and run the installer.
GitHub Desktop
  • Download GitHub Desktop from https://desktop.github.com/ and run the installer.
  • Click on "sign in to github.com".
(II.) First Run
Github Desktop:
  • On the start page, you see all repositories with granted access (access must be given by the owner of the corresponding repository first).
  • Clone e.g. github.com:mathebuddy/mathebuddy-public-courses or github.com:mathebuddy/mathebuddy-private-courses.
  • After cloning, click on "open in visual studio code".
  • In Visual Studio Code, click on "trust the authors" (if asked).
Visual Studio Code:
  1. Open the cloned repository (if not already done in the last step).
  2. Install the recommended extensions (must only be done once):
    1. Open the extensions menu on the left panel (the "blocks" icon; alternatively press [Ctrl]+[Shift]+[x]).
    2. In text field named by "search extensions in marketplace", type in "@recommended".
    3. Then install the recommended extensions (Spell-Checker + Python).
  3. Start the file server by either pressing [F5] or choosing [run] -> [debugging] in the main menu.
  4. You may grant access to Python to communicate within your local network.
  5. Check the installation: Open the address http://localhost:8271/ in your browser. If you see the directory listing of your course repository, then everything works as expected.
  6. Open https://mathebuddy.github.io/mathebuddy/ in your browser (Firefox was tested; other browsers may not yet work), click on [SIM], and then on the button [http://localhost:8271].
(III.) Daily Work
  • Edit courses with Visual Studio Code and run the file server with shortcut [F5].
  • Test your course data at https://mathebuddy.github.io on page [SIM], [http://localhost:8271].
  • Commit your own contributions with GitHub Desktop.
