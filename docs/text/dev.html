Course Development

Kursentwicklung

We offer the ability to create custom courses through an efficient workflow. The courses are written in the MatheBuddy Language (MBL) and compiled into a single file (MatheBuddy Compiled Language, MBCL) using the provided compiler. The simulator, currently available for macOS and Windows, renders the course data. VSCode is supported as an editor, including a plugin for syntax highlighting of MBL files. Wir bieten die Möglichkeit, benutzerdefinierte Kurse über einen effizienten Workflow zu erstellen. Die Kurse werden in der MatheBuddy-Language (MBL) verfasst und mit dem bereitgestellten Compiler in eine einzelne Datei kompiliert (MatheBuddy Compiled Language, MBCL). Der Simulator, derzeit für macOS und Windows verfügbar, stellt die Kursdaten dar. Als Editor wird VSCode unterstützt, inklusive eines Plugins für das Syntax-Highlighting von MBL-Dateien.

Quick GuideKurzanleitung

The following explains how to create custom course sections for the app. Im Folgenden wird erklärt, wie man eigene Kursteile für die App erstellen kann (nur Englisch).
  1. Install Python: https://www.python.org/downloads/
  2. Install VSCode: https://code.visualstudio.com/
  3. Open VSCode and install the plugin mbl-language-extension:
  4. Download the VSCode-project here and extract it (experts use git):
  5. In VSCode, select [File] → [Open] from the menu, and open the folder extracted in the previous step.

    The following screenshot shows the key work areas:
    Some hints:
    • After selecting the Explorer (E), you will see the file tree (F).
    • After selecting a file in (F), you will see the content in the text editor (T).
    • The course data is located in (F) in the course/ folder.
    • In the provided example, you can see the hierarchy: The folder complex/ contains a chapter. The chapters are listed in the course.mbl file. Each chapter consists of one or more units, which are specified in the index.mbl file within the chapter folder. A unit contains several levels, which are also located in the chapter folder as *.mbl files.
  6. This step is only necessary if you downloaded the project as *.zip-file in step 4 and you are using macOS as operating system (This issue does not occur when using the git command).
    • Open a terminal Window via the main menu: [Terminal] → [New Terminal]
    • In the just opened terminal, copy the following command and press the Enter-Key: xattr -d com.apple.quarantine tools/mathebuddy.app
  7. Before you can simulate your custom course, the course data must be compiled. There are two ways to start the compilation process:
    • Translate and simulate only the file currently open in the text editor (T).
    • Translate and simulate the entire course.
    You select one of the two options as follows:
    Then, the file or the entire course can be compiled via the main menu by selecting [Run] → [Start Debugging] (Tip: use the [F5] shortcut). Any errors will be displayed in the log area (L). If the log area is not visible, you can display it through the main menu by selecting [View] → [Terminal].
  8. Now, start the simulator through the Explorer/Finder of your operating system. It is located in the VSCode project that was downloaded and extracted in Step 4 of this guide.
    • On macOS, open mathebuddy.app in the tools-macos/ directory.
    • On Windows, open mathebuddy.exe in the tools-windows/app/ directory.
    Tip: It's best to place VSCode and the simulator side by side, as shown in the image at the top of this site.
  9. In the simulator, the file/course you just compiled should be visible. The simulator offers the following menu items, which are not available in the version distributed on the App Store:
    1. Updates the course data after restarting the compilation process in VSCode. This means you don't need to restart the simulator after every change.
    2. Switches between German and English language.
    3. Switches between Debug mode (D) and Release mode (R). In Debug mode, you get additional developer options, such as displaying the solution to a task on the on-screen keyboard, and more.
